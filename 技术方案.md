# 本地知识库系统技术方案

## 📋 项目概述

### 1.1 项目目标
构建一个基于 LangChain + RAG + Agent 架构的本地化知识库系统，满足以下需求：
- 支持个人及中小团队的文档管理和智能问答
- 实现完全本地化部署，保护数据隐私
- 支持中英文混合文献处理
- 提供双模式 LLM 支持（本地模型 + 云 API 后备）
- 集成智能 Agent 系统处理复杂任务

### 1.2 核心需求
- **文档管理**: 支持 PDF、DOCX、TXT、Markdown 等多格式文档
- **智能检索**: 基于语义的文档检索和问答
- **中文优化**: 针对中文文献的特殊处理
- **混合模式**: 本地模型为主，云 API 为后备
- **可视化界面**: 友好的 Web 操作界面
- **扩展性**: 支持功能模块扩展和定制

## 🎯 技术选型

### 2.1 本地大模型选型：DeepSeek-V2-Lite (16B MoE)

#### 选择理由：
1. **显存效率**: MoE 架构激活参数仅 ~2.1B，4-bit 量化后仅需 3-4GB 显存
2. **上下文长度**: 支持 64K tokens，适合长文档处理
3. **中英文能力**: 在中英文任务上表现均衡
4. **推理速度**: 激活参数少，推理速度较快
5. **Ollama 支持**: 官方支持，部署简单

#### 技术规格：
- **架构**: Mixture of Experts (MoE)
- **总参数**: 16B，激活参数: 2.1B
- **量化版本**: 4-bit Q4_K_M（质量损失小，显存占用少）
- **显存占用**: 3-4GB (RTX 4070 SUPER 12GB 完全足够)
- **上下文窗口**: 64K tokens

#### 备选方案：Qwen2.5-7B-Instruct
- **优势**: 中文能力略优，生态更成熟
- **劣势**: 稠密架构，显存占用较大 (4.5-5.5GB)
- **适用场景**: 如果 DeepSeek 中文表现不足时的备选

### 2.2 嵌入模型选型：BGE-M3

#### 选择理由：
1. **中文优化**: 专门为中文优化的嵌入模型
2. **多语言支持**: 支持中英文混合文本
3. **多功能**: 支持密集检索、多向量检索、稀疏检索
4. **效率**: 可在 CPU 上运行，节省 GPU 显存

#### 备选方案：
- **Nomic Embed Text**: 英文表现更好，上下文 8192
- **GTE-large**: 通用文本嵌入，质量稳定

### 2.4 向量数据库选型：ChromaDB

#### 选择理由：
1. **轻量级**: 本地部署简单，无需额外服务
2. **LangChain 集成**: 原生支持良好
3. **持久化**: 支持本地持久化存储
4. **功能完整**: 满足中小规模知识库需求

#### 扩展方案：
- **Qdrant**: 功能更丰富，支持高级检索特性
- **Weaviate**: 支持图神经网络，适合复杂关系
- **Pinecone**: 云服务，适合大规模部署

### 2.5 框架选型：LangChain + LangGraph

#### LangChain：
- **文档处理**: 丰富的文档加载器和处理器
- **RAG 流水线**: 内置完整的 RAG 组件
- **模型抽象**: 统一的模型接口，支持多后端
- **工具集成**: 丰富的工具和工具包

#### LangGraph：
- **Agent 编排**: 可视化的工作流设计
- **状态管理**: 内置状态管理和持久化
- **复杂任务**: 支持多步骤任务分解和执行
- **可观察性**: 完整的执行跟踪和调试

### 2.5 前端选型：Streamlit + FastAPI

#### Streamlit：
- **快速开发**: 用 Python 快速构建 Web 应用
- **交互组件**: 丰富的 UI 组件和布局
- **实时更新**: 支持实时数据更新和流式输出
- **部署简单**: 一键部署到 Streamlit Cloud

#### FastAPI：
- **高性能**: 基于 Starlette，性能优异
- **类型安全**: 基于 Pydantic 的自动验证
- **自动文档**: 自动生成 OpenAPI 文档
- **异步支持**: 完整的异步支持

#### 备选方案：Gradio
- **机器学习专用**: 为 ML 应用优化的组件
- **简单易用**: 更简单的 API 和布局
- **适合原型**: 快速构建演示原型

### 2.6 文档处理方案：MinerU + Markdown

#### 问题背景：
- PDF直接提取会丢失公式、图片
- 预处理会破坏LaTeX公式结构
- 需要专门工具处理含公式的科学文档

#### 选择理由：
1. **公式处理**: MinerU 能将 PDF 中的数学公式自动转为 LaTeX 格式
2. **表格识别**: 自动将表格转为 HTML 格式
3. **多语言 OCR**: 支持 109 种语言
4. **高准确性**: 针对科学文献优化

#### 转换流程：
```
PDF文档 → MinerU在线版 → Markdown(LaTeX公式) → 上传到知识库
```

#### 使用方式：
- **在线版（推荐）**: https://mineru.net/
  - 每天 2000 页免费额度
  - 无需配置，直接上传
- **API**: 配置 token 后可批量处理
  - Token有效期3个月

#### Token配置：
```bash
# 在 .env 文件中添加
MINERU_API_TOKEN=你的token
```

#### 系统改进：
- Markdown文件跳过破坏性预处理
- 使用二级分块（标题+大小）
- 公式边界保护（`$$` 和 `$`）

#### 3.2.1 双模式 LLM 管理器
```python
class DualModeLLMManager:
    """
    智能管理本地模型和云 API 的切换
    """
    
    def __init__(self, config):
        self.local_llm = OllamaLLM(model="deepseek-v2-lite:16b-q4_K_M")
        self.api_clients = {
            "openai": OpenAIClient(),
            "deepseek": DeepSeekClient(),
            "anthropic": AnthropicClient()
        }
        self.routing_rules = RoutingRules()
    
    def generate(self, prompt, context=None, **kwargs):
        # 1. 分析查询需求
        query_analysis = self.analyze_query(prompt, context)
        
        # 2. 选择最优提供商
        provider = self.select_provider(query_analysis)
        
        # 3. 调用相应服务
        if provider == "local":
            return self.local_llm.generate(prompt, **kwargs)
        else:
            return self.api_clients[provider].generate(prompt, **kwargs)
```

#### 3.2.2 RAG 流水线
```python
class RAGPipeline:
    """
    完整的 RAG 处理流水线
    """
    
    def __init__(self, config):
        self.document_loader = DocumentLoader()
        self.text_splitter = ChineseTextSplitter()
        self.embedder = BGE_M3_Embedder()
        self.vector_store = ChromaVectorStore()
        self.retriever = HybridRetriever()
        self.generator = RAGGenerator()
    
    def process_document(self, file_path):
        # 1. 加载文档
        documents = self.document_loader.load(file_path)
        
        # 2. 文本分割（中文优化）
        chunks = self.text_splitter.split_documents(documents)
        
        # 3. 生成嵌入
        embeddings = self.embedder.embed_documents(chunks)
        
        # 4. 存储到向量数据库
        self.vector_store.add_documents(chunks, embeddings)
        
        # 5. 返回处理结果
        return {"chunks_count": len(chunks), "status": "success"}
    
    def query(self, question, top_k=4):
        # 1. 检索相关文档
        retrieved_docs = self.retriever.retrieve(question, top_k)
        
        # 2. 重排序（可选）
        if self.config.enable_reranking:
            retrieved_docs = self.reranker.rerank(question, retrieved_docs)
        
        # 3. 生成答案
        answer = self.generator.generate(question, retrieved_docs)
        
        # 4. 返回结果
        return {
            "answer": answer,
            "sources": retrieved_docs,
            "retrieval_scores": [doc.score for doc in retrieved_docs]
        }
```

#### 3.2.3 Agent 系统
```python
class KnowledgeBaseAgent:
    """
    基于 LangGraph 的知识库 Agent
    """
    
    def __init__(self, config):
        self.workflow = StateGraph(AgentState)
        
        # 定义节点
        self.workflow.add_node("analyze_query", self.analyze_query)
        self.workflow.add_node("retrieve_documents", self.retrieve_documents)
        self.workflow.add_node("generate_answer", self.generate_answer)
        self.workflow.add_node("evaluate_answer", self.evaluate_answer)
        self.workflow.add_node("refine_answer", self.refine_answer)
        
        # 定义边
        self.workflow.set_entry_point("analyze_query")
        self.workflow.add_edge("analyze_query", "retrieve_documents")
        self.workflow.add_conditional_edges(
            "retrieve_documents",
            self.decide_next_step,
            {
                "generate": "generate_answer",
                "refine_query": "analyze_query"
            }
        )
        self.workflow.add_edge("generate_answer", "evaluate_answer")
        self.workflow.add_conditional_edges(
            "evaluate_answer",
            self.evaluate_quality,
            {
                "acceptable": END,
                "needs_refinement": "refine_answer"
            }
        )
        self.workflow.add_edge("refine_answer", "evaluate_answer")
        
        # 编译工作流
        self.app = self.workflow.compile()
    
    def invoke(self, query, **kwargs):
        # 执行 Agent 工作流
        result = self.app.invoke({
            "query": query,
            "max_iterations": kwargs.get("max_iterations", 5)
        })
        return result
```

## 🔧 性能优化策略

### 5.1 显存优化
1. **模型量化**: 使用 4-bit 量化 (Q4_K_M)
2. **分层加载**: 将嵌入模型放在 CPU，需要时加载到 GPU
3. **显存共享**: 使用内存映射文件减少重复加载
4. **动态卸载**: 长时间不用的模型层卸载到 CPU

### 6.1 智能切换策略
```python
class IntelligentRouter:
    """
    智能路由查询到最合适的 LLM 提供商
    """
    
    def route_query(self, query, context, user_preference=None):
        # 分析查询特征
        features = self.extract_features(query, context)
        
        # 应用路由规则
        if features["requires_realtime_info"]:
            return "api"  # 需要实时信息，使用 API
        
        if features["complexity"] > 0.8:
            return "api"  # 复杂推理，使用 API
        
        if features["sensitivity"] == "high":
            return "local"  # 隐私敏感，使用本地
        
        if features["context_length"] > 32000:
            return "api"  # 超长上下文，使用 API（如 Claude）
        
        # 默认使用本地
        return "local"
```

### 6.2 故障转移机制
```python
class FallbackManager:
    """
    故障转移和降级处理
    """
    
    def generate_with_fallback(self, prompt, provider="auto", max_retries=2):
        providers = self.get_provider_priority(provider)
        
        for i, provider_name in enumerate(providers):
            try:
                result = self.try_provider(provider_name, prompt)
                if result["success"]:
                    return result
            except Exception as e:
                logger.warning(f"Provider {provider_name} failed: {e}")
                
                if i == len(providers) - 1:
                    # 所有提供商都失败
                    raise Exception("All LLM providers failed")
                
                # 尝试下一个提供商
                continue
        
        # 如果所有都失败，返回降级响应
        return self.get_degraded_response(prompt)
```

### 6.3 成本控制
```python
class CostController:
    """
    API 使用成本控制
    """
    
    def __init__(self, budget_daily=10.0):  # 每日预算 10 美元
        self.budget_daily = budget_daily
        self.cost_today = 0.0
        self.usage_log = []
    
    def can_use_api(self, estimated_cost=0.01):
        """检查是否可以使用 API"""
        if self.cost_today + estimated_cost > self.budget_daily:
            logger.info("Daily budget exceeded, switching to local")
            return False
        return True
    
    def record_usage(self, provider, tokens, cost):
        """记录 API 使用和成本"""
        self.cost_today += cost
        self.usage_log.append({
            "timestamp": datetime.now(),
            "provider": provider,
            "tokens": tokens,
            "cost": cost
        })
```

```python
class QualityEvaluator:

    知识库系统质量评估
    """
    
    def evaluate_retrieval(self, query, retrieved_docs, ground_truth):

        metrics = {
            "recall@k": self.calculate_recall(retrieved_docs, ground_truth, k=5),
            "precision@k": self.calculate_precision(retrieved_docs, ground_truth, k=5),
            "ndcg@k": self.calculate_ndcg(retrieved_docs, ground_truth, k=5),
            "mrr": self.calculate_mrr(retrieved_docs, ground_truth)
        }
        return metrics
    
    def evaluate_generation(self, query, answer, ground_truth):
        """评估生成质量"""
        metrics = {
            "bleu": self.calculate_bleu(answer, ground_truth),
            "rouge": self.calculate_rouge(answer, ground_truth),
            "bertscore": self.calculate_bertscore(answer, ground_truth),
            "faithfulness": self.evaluate_faithfulness(answer, retrieved_context),
            "relevance": self.evaluate_relevance(answer, query)
        }
        return metrics
```
