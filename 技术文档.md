## 1. 项目概述

### 1.1 项目目标

基于 LangChain + RAG + Agent 架构的本地知识库系统，实现：
- 文档智能管理与问答
- 完全本地化部署，保护数据隐私
- 中英文混合文献处理
- 双模式 LLM 支持（本地模型 + 云 API）
- MinerU 文档导入自动化

### 1.2 核心特性

| 特性 | 说明 |
|------|------|
| **双模式 LLM** | DeepSeek-V2-Lite 本地模型 + API 后备 |
| **完整 RAG** | 文档处理 → 向量化 → 检索 → 生成 |
| **中文优化** | 针对中英文混合文献的特殊处理 |
| **MinerU 集成** | 自动导入 PDF 转换的 Markdown |
| **元数据管理** | 自动提取标题、作者、摘要 |
| **图片处理** | 自动关联文档图片 |

## 3. 核心模块

### 3.1 配置管理 (`src/core/config.py`)

```python
from src.core.config import get_config

# 获取配置
model = get_config("llm.local.ollama.model", "deepseek-v2:lite")
chunk_size = get_config("document_processing.chunking.chunk_size", 800)
```

**配置项** (`config/settings.yaml`):
- `llm.local.*` - 本地模型配置
- `llm.api.*` - API 模型配置
- `document_processing.*` - 文档处理配置
- `rag.*` - RAG 检索配置
- `paths.*` - 路径配置

### 3.2 文档处理 (`src/core/document_processor.py`)

**功能**:
- 多格式支持 (PDF, TXT, DOCX, MD, CSV, HTML)
- 中文优化分块
- Markdown 格式保护
- LaTeX 公式保护
- 元数据自动提取

**核心类**:
```python
from src.core.document_processor import DocumentProcessor

processor = DocumentProcessor()
documents = processor.process_file("path/to/document.md")
```

**ChineseTextSplitter** - 中文优化分块器：
- 按段落、句子分割
- 保护 Markdown 格式标记
- 保护 LaTeX 公式

### 3.3 向量存储 (`src/core/vector_store.py`)

```python
from src.core.vector_store import SimpleVectorStore

vs = SimpleVectorStore()
vs._ensure_initialized()
vs.add_documents(documents)
results = vs.similarity_search(query, k=4)
```

### 3.4 LLM 管理器 (`src/core/llm_manager.py`)

**双模式支持**:
- **local**: Ollama 本地模型
- **api**: DeepSeek/OpenAI/Kimi/Claude API

```python
from src.core.llm_manager import LLMManager

llm = LLMManager()
result = llm.generate(prompt, provider="local")  # 或 "api"
```

### 3.5 MinerU 导入器 (`src/utils/mineru_importer.py`)

**功能**:
- 自动读取 MinerU 输出目录
- 提取 full.md 内容
- 生成友好文件名
- 复制文档到 raw_docs
- 复制图片到 images 目录
- 自动提取元数据

```python
from src.utils.mineru_importer import import_mineru_document

result = import_mineru_document("C:/Users/.../MinerU/xxx-hash")
# 返回: filename, title, authors, has_images 等
```

## 4. 数据结构

### 4.1 目录结构

```
data/
├── raw_docs/              # 原始文档
│   └── *.md
├── images/                # 文档图片（按文档名分类）
│   ├── 文档名_images/
│   │   └── images/
│   │       └── *.jpg
├── metadata/              # 元数据
│   └── 文档名.meta.json
└── vector_store/         # ChromaDB 向量存储
    └── chroma/
```

### 4.2 元数据格式

```json
{
  "title": "文档标题",
  "authors": ["作者1", "作者2"],
  "date": "2024-11",
  "document_type": "paper",
  "field": "计算机科学",
  "keywords": ["AI", "RAG"],
  "abstract": "文档摘要内容..."
}
```

### 4.3 Chunk 元数据

每个向量 chunk 包含:
```python
{
    "source": "文件完整路径",
    "format": "md",
    "file_name": "文件名.md",
    "title": "文档标题",
    "authors": [...],
    "abstract": "...",
    "document_type": "paper"
}
```

## 5. API 接口

### 5.1 健康检查

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/health` | 整体健康检查 |
| GET | `/health/local` | 本地 Ollama 状态 |
| GET | `/health/api` | API 密钥状态 |
| GET | `/health/vectorstore` | ChromaDB 状态 |

### 5.2 文档管理

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/v1/documents/list` | 文档列表 |
| GET | `/api/v1/documents/stats` | 文档统计 |

| POST | `/api/v1/documents/process` | 处理/向量化 |
| DELETE | `/api/v1/documents/{filename}` | 删除文档 |
| GET | `/api/v1/documents/{filename}/chunks` | 获取 chunks |

### 5.3 问答

| 方法 | 路径 | 说明 |

  "top_k": 4

## 7. 使用指南

### 7.1 快速启动

```bash
# 方式1: 使用启动脚本
start.bat

# 方式2: 手动启动
# 终端1: 启动 API
python src/api/main.py

# 终端2: 启动前端
streamlit run src/frontend/app.py
```

### 7.2 添加文档

**方式1: 直接上传**
1. 选择"直接上传"
2. 选择文件
3. 点击"处理文档"

**方式2: MinerU 导入**
1. 选择"从 MinerU 导入"
2. 点击"选择文件夹"
3. 选择 MinerU 输出目录
4. 自动完成导入和向量化

### 7.3 元数据

系统会自动提取元数据：
- 无 .meta.json → 调用本地 LLM 提取
- 有 .meta.json → 直接读取

**手动创建元数据**:
```json
// data/metadata/文档名.meta.json
{
  "title": "论文标题",
  "authors": ["作者1", "作者2"],
  "keywords": ["关键词1", "关键词2"]
}
```

## 9. 常见问题

### Q1: 启动时显示"API服务未连接"
- 确认 `start.bat` 已运行
- 等待 Ollama 完全启动
- 右上角会自动检测并刷新

### Q2: 图片不显示
- 检查 `data/images/文档名_images/` 目录
- 确认图片文件存在

### Q3: 元数据提取失败
- 确认 Ollama 服务正常运行
- 检查本地模型是否安装: `ollama list`

### Q4: 向量化速度慢
- 首次运行需要初始化嵌入模型
- 可在侧边栏点击"🔥 预热模型"
