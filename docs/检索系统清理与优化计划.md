# 检索系统清理与优化计划

## Context

当前系统存在两个核心问题：
1. **API 绕过了 RAGChain**：`/api/v1/query` 端点直接调用 `vector_store.similarity_search()`，完全没用混合搜索、父子检索、重排序等高级功能。RAGChain 和所有高级检索代码在生产中实际是死代码。
2. **存在大量无用配置和机制**：多个 config 项从未被读取、多个工具/函数从未被调用、检索策略互斥无法组合。

本计划目标：清理无用代码，修复 bug，让 API 走 RAGChain 完整检索管线，并增强检索质量。

## Phase 2: 修复 Bug

### 2.1 修复 RRF 文档去重 key 碰撞

## Phase 3: API 接入 RAGChain（核心改动）

### 3.1 扩展 RAGChain.query() 签名
**文件**: `src/core/rag_chain.py`
```python
def query(
    self, question: str,
    history: Optional[List[Dict[str, str]]] = None,
    provider: Optional[str] = None,
    top_k: Optional[int] = None,
) -> Dict[str, Any]:
```
- `history` 传入后写入 prompt 历史区段
- `provider` 透传给 `llm_manager.generate(prompt, provider=provider)`
- `top_k` 覆盖默认 `retrieval_top_k`
- `use_history` 参数删除（从未实现）

### 3.2 移植 API 的优秀 Prompt 到 RAGChain
**文件**: `src/core/rag_chain.py`, `_build_prompt()`
- 移植 API 中更强的反幻觉约束（"禁止幻觉"、严格引用规则）
- 移植短事实检测（数字/时间类问题用精简 prompt）
- 移植历史对话构建逻辑（最近6轮）
- RAGChain 成为 prompt 构建的唯一来源

### 3.3 API 端点委托给 RAGChain
**文件**: `src/api/main.py`
- 添加 `get_rag_chain()` 单例（线程安全，与现有 get_* 模式一致）
- `/api/v1/query` 的 RAG 分支改为调用 `rag_chain.query()`
- 删除 API 中的重复逻辑（`_build_prompt_with_history`、`_build_history_context`）
- 保留 `use_rag=False` 的直接生成分支

## Phase 4: 增强检索质量

### 4.1 检索策略可组合
**文件**: `src/core/rag_chain.py`, `_retrieve_documents()`
- 重写 if-elif 链为 pipeline：先选检索方式 → 再过 reranking → 再截断到 top_k
- reranking 已在 `query()` 中独立调用，不需要改动
- 检索阶段用 `fetch_k = top_k * 3` 过量检索，reranking 截断回 top_k

### 4.2 BM25 加入 jieba 中文分词
**文件**: `src/core/vector_store.py`, `BM25._tokenize()`
- 中文文本用 `jieba.cut()` 分词，英文用空格
- jieba 未安装时 fallback 到现有正则
**文件**: `pyproject.toml` + `requirements.txt`
- 添加 `jieba>=0.42.1` 依赖

### 4.3 BM25 读取配置参数
**文件**: `src/core/vector_store.py`
- `_get_bm25_index()` 和 `hybrid_search()` 中创建 BM25 实例时读取 `settings.yaml` 的 `bm25_k1`/`bm25_b`

### 4.4 跨语言翻译加缓存
**文件**: `src/core/vector_store.py`, `QueryTranslator`
- 添加 `Dict[str, str]` 内存缓存，key 为 `target_lang:query`
- 避免同一查询重复调 LLM 翻译

### 4.5 API 查询端点支持检索模式选择
**文件**: `src/api/main.py`
- `QueryRequest` 新增可选字段 `retrieval_mode: Optional[str]`，默认 None（用配置）
- 值为 `"hybrid"`, `"parent_child"`, `"cross_lingual"` 时覆盖 RAGChain 的检索策略

## 验证方式

1. `pytest tests/ -v` — 确保所有现有测试通过
2. 启动 API：`python src/api/main.py`
3. 测试查询端点：
   - `curl -X POST http://localhost:8000/api/v1/query -H "Content-Type: application/json" -d '{"question":"测试问题","use_rag":true}'`
   - 验证日志中出现 "使用混合搜索"（而非直接 similarity_search）
4. 检查 BM25：上传新文档后执行查询，确认 BM25 索引重建（缓存失效）
