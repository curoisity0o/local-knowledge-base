## 📋 项目概述

基于 LangChain + RAG + Agent 架构的本地知识库系统，具有以下核心特性：

- **双模式LLM**: DeepSeek-V2-Lite 本地模型 (16B MoE, 4-bit量化) + 大模型API后备
- **完整RAG流水线**: 文档处理 → 向量化 → 检索 → 增强生成
- **中文环境优化**: 针对中英文混合文献的特殊处理
- **多界面支持**: Streamlit 可视化界面 + FastAPI REST API
- **硬件适配**: 针对 i5-13600KF, 32GB RAM, RTX 4070 SUPER 12GB 优化

## ⚠️ 已知问题与解决方案

### 1. 嵌入模型下载超时
**问题**: BAAI/bge-m3模型从HuggingFace下载可能超时
**解决方案**:
   - 使用国内镜像源：`pip config set global.index-url https://pypi.tuna.tsinghua.edu.cn/simple`
   - 使用轻量级本地模型：`text2vec-chinese` (约400MB)
   - 预先下载模型到本地目录

### 2. 本地模型依赖
**问题**: 需要Ollama服务运行DeepSeek-V2-Lite
**解决方案**:
   - 安装Ollama Windows版
   - 下载4-bit量化模型减少显存占用
   - 备选方案：使用LLaMA-Factory（现有目录 `D:\code\LLM\LLaMA-Factory`）

### 3. 硬件配置适配
**硬件**: i5-13600KF, 32GB RAM, RTX 4070 SUPER 12GB
**优化建议**:
   - DeepSeek-V2-Lite (4-bit): 预计占用8-10GB显存
   - 批处理大小：建议4-8（根据显存调整）
   - 并行处理：CPU核心充足，可并行处理文档

## 📞 技术支持

### 关键配置文件
- **主配置**: `config/settings.yaml`
- **模型配置**: `config/models.yaml`
- **环境变量**: `.env.example`（复制为`.env`并填写）

### 测试入口
- **组件测试**: `python src/main.py`
- **API服务**: `uvicorn src.api.main:app --reload`
- **前端界面**: `streamlit run src/frontend/app.py`

### 故障排查
1. **导入错误**: 确保安装所有依赖 `pip install -r requirements.txt`
2. **编码问题**: 设置环境变量 `PYTHONUTF8=1`
3. **模型下载**: 使用国内镜像源或预下载模型
4. **显存不足**: 调整批处理大小，使用4-bit量化
